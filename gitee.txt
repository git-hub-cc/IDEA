Gitee 的 **Client ID** 和 **Client Secret** 是在你注册 OAuth 应用时生成的，用于授权第三方应用访问 Gitee 用户的资源（如获取用户信息、提交代码等）。它们一般用于 **OAuth2 授权流程**。

---

## 🧩 一、获取方式

你需要先在 Gitee 注册 OAuth 应用：

1. 访问 Gitee 的开发者后台：[https://gitee.com/oauth/applications](https://gitee.com/oauth/applications)
2. 点击 “注册应用”
3. 填写应用信息，如：

   * 应用名称
   * 应用主页 URL
   * 授权回调地址（如：`http://localhost:8080/callback`）
4. 创建后即可获得：

   * **Client ID**
   * **Client Secret**

---

## 🔐 二、如何使用（标准 OAuth2 流程）

以下是使用 `Client ID` 和 `Client Secret` 进行 **OAuth2 授权码流程（Authorization Code Flow）** 的步骤。

---

### 步骤 1：引导用户授权

你需要将用户引导到 Gitee 授权页面：

```
https://gitee.com/oauth/authorize?client_id=你的ClientID&redirect_uri=你的回调地址&response_type=code
```

示例：

```url
https://gitee.com/oauth/authorize?client_id=abc123&redirect_uri=http://localhost:8080/callback&response_type=code
```

用户同意授权后，Gitee 会重定向回你设置的 `redirect_uri`，并携带一个参数 `code`。

---

### 步骤 2：使用 code 获取 access\_token

你服务器用 `code` 去 Gitee 的接口换取 `access_token`：

**POST 请求** 到：

```
https://gitee.com/oauth/token
```

**请求参数（application/x-www-form-urlencoded）：**

| 参数             | 说明                       |
| -------------- | ------------------------ |
| grant\_type    | `authorization_code` 固定值 |
| code           | 上一步获得的授权码                |
| client\_id     | 你的Client ID              |
| redirect\_uri  | 与注册应用时一致                 |
| client\_secret | 你的Client Secret          |

示例 curl 请求：

```bash
curl -X POST https://gitee.com/oauth/token \
  -d "grant_type=authorization_code" \
  -d "code=abcxyz" \
  -d "client_id=你的ClientID" \
  -d "redirect_uri=http://localhost:8080/callback" \
  -d "client_secret=你的ClientSecret"
```

返回结果：

```json
{
  "access_token": "xxxx",
  "token_type": "bearer",
  "expires_in": 86400,
  "refresh_token": "xxxx"
}
```

---

### 步骤 3：用 access\_token 访问 Gitee API

比如获取用户信息：

```bash
curl https://gitee.com/api/v5/user?access_token=你的access_token
```

---

## 🧪 测试建议

* 在开发阶段，可以将 `redirect_uri` 设置为本地地址（如 `http://localhost:8080/callback`）
* 确保你服务器能处理回调并提取 `code` 参数
* 所有请求推荐使用 HTTPS
